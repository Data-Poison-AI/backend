# Risk Engine

The Risk Engine is responsible for aggregating the outputs generated by the different detection modules and producing a final risk assessment for each analyzed input.

Instead of relying on a single detector, the system combines the results from multiple modules to determine the overall threat level. This approach improves reliability and reduces the probability of false positives or false negatives.

The Risk Engine ensures that the decision process remains transparent and explainable rather than operating as a black-box system.


## Input Sources

The Risk Engine receives risk indicators from the different detection modules implemented in the system. Each module analyzes the input and produces a score representing the likelihood of a malicious pattern.

The system currently integrates results from the following detectors:

- Prompt Injection Detector

- Data Poison Detector

- Backdoor Detector

- Steganography Detector

Each detector returns a score that represents the confidence level of a potential threat detection.

## Score Normalization

Since different detectors may produce outputs on different scales, the Risk Engine first performs a normalization step.

All detector scores are converted into a common range between 0 and 100, where:

- 0 represents no detected risk
- 50-70 represents suspicious
- 70-100 represents maximum risk

This normalization ensures that the outputs from different modules can be compared and combined consistently.

## Weighted Risk Formula

After normalization, the system calculates the final risk score using a weighted aggregation model.

Each detector contributes to the final score based on its relative importance in the detection process.

Example weighted formula:

### Final Risk Score =
- (0.25 × PromptInjectionScore) +
- (0.25 × DataPoisonScore) +
- (0.25 × BackdoorScore) +
- (0.25 × SteganographyScore)

This approach allows the system to prioritize certain threats while still considering all available signals.

## Design Decisions

Several design principles guided the development of the Risk Engine:

### Transparency

The scoring system is designed to remain explainable. Each detector contributes to the final result, making it possible to trace how the risk score was calculated.

### Modularity

The architecture allows additional detection modules to be integrated without modifying the overall system logic.

### Flexibility

Weights assigned to detectors can be adjusted depending on the operational environment or the type of AI system being protected.

## Conclusion

The Risk Engine plays a central role in the system by combining multiple detection signals into a single, interpretable risk score. This aggregation process enables the system to make consistent and transparent decisions when identifying potentially malicious AI inputs.
